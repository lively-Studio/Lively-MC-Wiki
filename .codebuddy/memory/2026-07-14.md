## 子文档路由修复 (2026-07-14)
- **问题**：87 个 `docs/blocks/*.md` 文档无法从侧边栏或 `blocks/index.md` 中打开。原因：
  1. `config.json` 的 `/blocks` 路由映射到 `blocks.md`（总结表），而非 `blocks/index.md`（含 87 个链接的导航页）
  2. `app.js` 路由系统不支持 `/blocks/stone` 这种子路径，`ROUTES` 中没有对应条目
  3. link click handler 只取文件名（如 `stone.md`），丢失了目录上下文
- **修复**（`app.js` 3 处改动 + `config.json` 1 处改动）：
  1. `config.json`: `/blocks` 路由的 file 改为 `blocks/index.md`
  2. `loadDocument()`: 新增子路径自动解析逻辑，当路由如 `/blocks/stone` 不在 ROUTES 中时，自动查找父路由并构建文件路径 `blocks/stone.md`，同时注册到 ROUTES 和 FILE_TO_ROUTE 以便后续查找
  3. link click handler: 使用 `CURRENT_ROUTE` + `ROUTES[currentRoute].file` 推断当前文档所在目录，为相对链接（如 `stone.md`）补全目录前缀
  4. `handleRoute()`: 增加 `hash.includes('/')` 条件，让 `blocks/stone` 格式的 hash 触发文档加载
  5. 新增 `CURRENT_ROUTE` 全局变量追踪当前路由
