# 红石火把

![Redstone Torch](https://minecraft.wiki/images/Redstone_Torch_JE3_BE2.png?format=original)

> **稀有度**：常见 | **可再生**：是 | **可堆叠**：64 | **工具**：任何
> **硬度**：0 | **爆炸抗性**：0 | **发光**：是（7，激活时） | **透明**：否 | **可燃**：否
> **Java ID**：`redstone_torch` | **基岩 ID**：`redstone_torch` (76)
> **地图颜色**：`#FF0000`

红石火把是红石系统的核心元件之一，它既是**信号源**（持续输出信号 15），也是一个天然的反相器（**NOT 门**）。当它附着的方块被充能时，红石火把会**熄灭**——从而实现逻辑"非"操作。

---

## 获取

### 合成
```
1 红石粉 + 1 木棍

  红石粉
  木棍
```

### 自然生成
- **雪屋地下室**：墙壁上
- **古迹废墟**：机关中

### 破坏
- 任何工具均可挖掘，立即掉落

---

## 行为

### 作为信号源
- 持续输出 **信号强度 15**
- 可向上方和相邻方块输出信号
- 可充能**上方的实心方块**（间接激活相邻元件）

### 反相器（NOT 门）
```
方块A ← 输入信号 ON
  └─[红石火把] ← 熄灭（输出 OFF）

方块A ← 输入信号 OFF
  └─[红石火把] ← 亮起（输出 ON）
```
- 附着的方块被充能 → 红石火把熄灭
- 附着方块不被充能 → 红石火把亮起

---

## 烧毁保护

- 红石火把在 **8 游戏刻（0.4 秒）内开关超过 8 次** → 自动"烧毁"
- 烧毁的红石火把变暗，停止工作
- 必须**手动更新**（破坏并重新放置或更新相邻方块）才能恢复
- 设计高速电路时需注意避免

---

## 经典应用

### NOT 门
```
输入 → 方块 → 红石火把 → 输出（反相）
```

### 垂直信号传输
- 红石火把 + 方块塔 = 向上传输信号

### 时钟电路
```
红石火把 → 红石粉回路 → 方块 → 红石火把 → ...
```
- 经典的火把时钟

## 你知道吗

1. 🔄 红石火把是 Minecraft 中唯一既是**光源又是逻辑门**的方块——它天然实现 NOT 运算。
2. ⚡ 火把烧毁机制防止了无限循环——8 刻内 8 次以上开关自动保护。
3. 💡 红石火把亮度为 7，不足以阻止怪物生成（需要亮度 > 7）。
4. 🏗️ Java 版中红石火把分为 `redstone_torch`（地上）和 `redstone_wall_torch`（墙上）两个独立方块。
